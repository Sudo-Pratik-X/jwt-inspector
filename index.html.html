Quick Reference

Recommended: HS256/384/512 (shared secret), RS256/384/512 & ES256/384/512 (public/private key pairs).

Avoid: alg: none (no signature at all — critical if your verifier accepts it), and never trust the alg header from an untrusted token without pinning the expected algorithm server-side.

Always check: exp (expiry), aud (intended recipient), iss (issuer), and nbf (not-before) — a valid signature alone doesn't mean a token should be accepted.
